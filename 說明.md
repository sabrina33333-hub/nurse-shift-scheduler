         # 護理師排班系統 - 開發說明文件（v1.0）

## 專案概述

Java Maven 專案，開發護理師自動排班系統。

| 項目 | 設定值 |
|------|--------|
| Java 版本 | 17 |
| 依賴 | `poi-ooxml` v5.2.3（Excel 匯出） |
| 原始碼路徑 | `src/main/java/shiftSystem/` |
| 病房人數 | 25 人 |
| 排班週期 | 月為單位 |
| 使用者角色 | 護理長 (HEAD) / 護理師 (NURSE) |

---

## 班別定義

| 代號 | 班別名稱 | 時間 | 性質 |
|------|----------|------|------|
| `D` | 白班 | 08:00 ~ 16:00 | 主要班別 |
| `E` | 小夜 | 16:00 ~ 24:00 | 主要班別 |
| `N` | 大夜 | 24:00 ~ 08:00 | 主要班別 |
| `M` | 10-6 中間班 | 10:00 ~ 18:00 | 彈性班（人力調節用） |
| `OFF` | 公休 | - | 一般休假 |
| `*OFF` | 預假 | - | 預先排定的休假 |
| `H_OFF` | 國定假日休 | - | 國定假休假 |

目前 `ShiftType` enum 僅實作 `DAY / EVENING / NIGHT / NONE`，`M / OFF / *OFF / H_OFF` 待新增。

---

## 排班鐵則

| 編號 | 規則 | 嚴重度 |
|------|------|--------|
| R1 | 不能「大夜接白班」 | 強制 |
| R2 | 不能「小夜接白班」 | 強制 |
| R3 | OFF 前後不能夾 1 天班 | 警告 |
| R4 | 連續上班最多 6 天 | 強制 |
| R5 | 每班至少 1 位資深護理師 | 強制 |
| R6 | 每班最少人力：白 ≥ 3 / 小夜 ≥ 2 / 大夜 ≥ 1 | 強制 |
| R7 | 換班後仍須符合 R1~R6 | 強制 |
| R8 | 白班、小夜班不能接大夜班

**實作難易度（由易到難）：** R6 → R5 → R3 → R4 → Excel 匯出 → R7



---

## 休假優先順序

```
國定假日 > 一般假日 > 個人預排公休
```

當多人想休同一天且超過可休人數上限 → 同層級申請進入抽籤（見模組 C）。

## 模組 A：積假計算系統

護理師在國定假日上班 → 自動累積積假時數 → 未來可申請補休。

### 資料模型

```java
class LeaveRecord {
    String nurseId;
    String date;           // 累積日期
    double hours;          // 時數（例如 8 小時）
    String type;           // "ADD" 累積 / "USE" 使用
    String reason;         // 例：「2026-05-01 勞動節上班」
    String status;         // "PENDING" / "APPROVED"
}

class LeaveBalance {
    Member nurse;
    double overtimeBalance;             // 積假時數餘額
    ArrayList<LeaveRecord> records;     // 所有加班 / 使用紀錄

    void addOvertime(double hours);     // 累積積假
    void useLeave(double hours);        // 使用積假
    double getCurrentBalance();         // 查詢餘額
}
```

### 功能流程

1. 護理師上國定假日的班 → 系統自動 +8 小時積假
2. 護理師提出「使用積假」申請
3. 護理長審核 → 核准後扣除時數
4. 隨時可查詢目前積假餘額

## 模組 B：智慧代班媒合系統

突發請假時，自動篩選符合條件的代班人選。

### 篩選邏輯

```
代班人選池 = 所有護理師
  ↓ 過濾 1：那天原本是 OFF 的人
  ↓ 過濾 2：isAvailableForSub == true（願意被通知）
  ↓ 過濾 3：availableSubShifts 包含該班別
  ↓ 過濾 4：代班後不違反 R1~R6
  ↓ 排序：優先推薦本月代班次數最少的人（公平原則）
  ↓
顯示前 5 位推薦代班人選
```

### 資料模型

```java
class BackupRequest {
    String requestId;
    Member requester;                   // 請假人
    String date;                        // 請假日期
    String shift;                       // 哪個班
    String reason;                      // 原因
    String status;                      // "PENDING" / "ACCEPTED" / "REJECTED"

    ArrayList<Member> candidates;       // 候選人名單
    Member acceptedBy;                  // 最終接班人
}
```

### 工作流程

```
1. 護理師按「我要請假」→ 選日期、填原因
2. 系統從其 backupCandidates 名單抓出候選人，過濾不符條件的
3. 發送代班邀請給剩餘候選人
4. 第一個按「我可以」的人 → 自動換班成功
   若全部拒絕 → 通知護理長處理
5. 班表自動更新，雙方收到確認
```

## 模組 C：抽籤系統（多人搶同一天休假）

### 觸發條件

多位護理師申請同一天公休，且當天人力不允許全部批准時啟動。

### 抽籤流程

```
1. 蒐集同一天、同優先級的休假申請清單
2. 計算當天剩餘可休人數（總人力 - 最小班別需求）
3. 若申請人數 ≤ 可休人數 → 全部批准，不需抽籤
4. 若申請人數 > 可休人數 → 進入抽籤
   - 從申請清單隨機抽出「可休人數」位護理師 → 批准
   - 其餘 → 拒絕並通知
5. 護理長可查看抽籤結果並手動覆寫（最終審核權在護理長）
```

### 資料模型

```java
class LotteryRequest {
    String date;                        // 抽籤日期
    String priority;                    // "HOLIDAY" / "GENERAL" / "PREFERRED"
    ArrayList<String> applicants;       // 申請護理師 id 清單
    int availableSlots;                 // 可批准人數
    ArrayList<String> winners;          // 中獎名單
    ArrayList<String> losers;           // 落選名單
    String status;                      // "PENDING" / "DONE"
}
```

### 注意事項

- 抽籤結果需寫回班表（中獎者設為 OFF，落選者維持原班）
- 抽籤後班表變動需重新通過 R7 驗證
